建站感受

Thu, Dec 12, 2019

为什么要搭建这个网站？

看过很多博客，主要是面向程序员的，大部分博客第一篇文章都是关于为什么这个主题，很多人都是为了生活、技术和成长中的一些经历；后面又听到一个观点，“做一件事情最好的时机是3年前，其次是现在”，由此，确定要搭建属于自己的博客。

可能刚开始主要是写给自己看的把，毕竟属于摸着石头过河的阶段，质量和数量都不会太多，篇幅也不会太长。

输出是最好的输入

在程序员这个圈子里，一个好的博客是一个非常能够体现程序员水平和能力的名片，而且，是一个表达自己思想和分享自己知识的一个绝佳的平台，这也非常符合这个圈子的文化。

在学习一项技术的时候，面向过程和结果来看，博客都是一个很好的平台，学习是一个输入的过程，博客作为一个输出的过程刚好能够实现完整的输入输出闭环网络，能够准确表达的知识才是内化于心，属于自己的知识。

技术栈

这次建站用到的相关技术主要有GitHub Pages托管服务，hugo博客发布工具：

  • GitHub Pages
    • 静态站点托管服务
    • 支持自定义域上托管站点
    • 支持https
  • hugo

    我就是按照中文社区首页的步骤一步步搭建的

    • Go语言实现
    • 简单、易用、高效
    • 快速部署

总结

这次建站总体来说是一次全新的体验，以前一直在纠结要在哪里去搭建博客，用什么工具来搭建博客，终于找到一个理想的平台，不用备案，不用自购主机，域名可以自主选择绑定，一切都很好。

搭建过程中，没多少纠结就选择了hugo作为发布工具，倒是在使用theme的时候遇到了较大的麻烦，最终还是通过不断尝试和主动搜索解决问题，总结几点：

  • theme里面会有一套默认的配置，hugo根目录有一组空的配置，好的做法是拷贝一份theme的配置到hugo根目录，就可以随心所欲自定义了
  • 主动去阅读命令行返回的信息，快速解决问题；可以避免一些简单的问题，减少搜索的次数和时间
  • 自定义theme中有很多的css相关的知识

至此，一个基本的博客框架基本完成，后续还有一些自定义功能还需完善，例如字数统计、标签分类等，现在就主要的就是内容了，毕竟“好看的皮囊千篇一律，有趣的灵魂万里挑一”。
